## 2. `pyproject.toml` (pinned)

Pins derived from upstream pyprojects (`serena/pyproject.toml`, `mempalace/pyproject.toml`) and from the MCP SDK minor most compatible with both.

```toml
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[project]
name = "engram"
version = "0.1.0"
description = "Unified coding-agent substrate: symbol + memory + vector, with a Link Layer."
readme = "README.md"
license = "MIT"
requires-python = ">=3.11,<3.15"

dependencies = [
    # MCP wire — match Serena's pin; MemPalace supports 1.x too
    "mcp==1.27.0",

    # Upstreams installed as Python wheels for in-proc fact_checker import + process management
    "serena-agent==1.1.2",
    "mempalace==3.3.3",

    # Engram's own runtime
    "pydantic==2.12.5",
    "click>=8.1,<9",
    "watchdog>=4.0,<5",
    "pyyaml==6.0.2",
    "aiosqlite>=0.20,<1",
    "anyio>=4.4,<5",
]

[project.optional-dependencies]
dev = [
    "pytest>=8.2,<9",
    "pytest-asyncio>=0.23,<1",
    "ruff>=0.5,<1",
    "mypy>=1.10,<2",
]

[project.scripts]
engram = "engram.cli:main"
engram-mcp = "engram.server:main"

[tool.ruff]
line-length = 100
target-version = "py311"
```

**Note on claude-context.** It is a Node package, not a Python wheel. Engram does not list it in `dependencies`; instead, `engram init` checks for `node >= 20 < 24` and does a `npm install -g @zilliz/claude-context-mcp@0.1.8` or documents `npx @zilliz/claude-context-mcp@0.1.8`. Version pinning in `engram.config.default.upstreams.claude_context_version`.

## 3. `.engram/config.yaml` schema

One config file per workspace, generated by `engram init`. Pydantic model in `src/engram/config.py`.

```yaml
# .engram/config.yaml — generated by `engram init`; safe to commit.
version: 1

workspace:
  root: .                    # resolved to absolute path at runtime
  name: my-project

anchors:
  db_path: .engram/anchors.sqlite
  reconcile_interval_hours: 24
  wal_tailer_poll_ms: 500

upstreams:
  serena:
    command: ["serena", "start-mcp-server"]
    args_extra: []                       # appended per-invocation
    working_dir: .                       # inherits workspace.root by default
  mempalace:
    command: ["mempalace-mcp"]
    palace_path: ~/.mempalace/palace
    wal_path: ~/.mempalace/wal/write_log.jsonl
  claude_context:
    command: ["npx", "@zilliz/claude-context-mcp@0.1.8"]
    embedding_provider: Ollama           # or OpenAI, VoyageAI, Gemini, OpenRouter
    embedding_model: nomic-embed-text
    milvus_address: localhost:19530
    snapshot_path: ~/.context/mcp-codebase-snapshot.json
    merkle_path: ~/.context/merkle       # directory
    reindex_tick_seconds: 300            # 5 min; doc 06 §5.3

router:
  fusion:
    algorithm: rrf
    k: 60
  cache:
    ttl_seconds:
      vec_search: 60
      code_find_symbol: 30
      mem_search: 120
      mem_kg_query: 300
      fused: 60
    max_entries: 1024

logging:
  level: INFO
  file: .engram/logs/engram.log
```

**Validation.** Pydantic rejects unknown keys; `engram init` stamps the version field and `engram` commands refuse to start against a config from a newer version than they understand.

## 4. CLI surface

```
engram init [--workspace DIR] [--embedding-provider PROVIDER]
   Create .engram/config.yaml; assert prerequisites (Python >=3.11, Node >=20<24,
   Serena + MemPalace + claude-context-mcp reachable); initialize the anchor DB.

engram mcp [--port N]
   Start the Engram MCP server on stdio (default) or a TCP port. This is what the
   user agent connects to.

engram smoke-test [--workspace DIR]
   Programmatic end-to-end check: starts the three upstreams, calls engram.health,
   creates a sample anchor, tears down. Returns exit 0 on green, 1 on red.

engram reconcile [--scope {symbols,chunks,memories,all}]
   Invoke engram.reconcile outside the daily schedule.

engram logs [--tail N] [--level LEVEL]
   Tail Engram + upstream logs, merged with timestamps.

engram status
   Print the output of engram.health in a human-readable table.
```

All commands return structured JSON with `--json` for agent consumption.

## 5. MCP client registration — JSON snippets

Engram is registered with a user agent as a single MCP server. The three upstreams remain invisible to the user's agent.

### Claude Code

Reference: `docs.anthropic.com/en/docs/claude-code/mcp` (user's own Claude Code docs).

```bash
claude mcp add engram \
  --scope user \
  -- engram-mcp
```

Or edit `~/.claude.json` manually:

```json
{
  "mcpServers": {
    "engram": {
      "command": "engram-mcp",
      "args": [],
      "env": {
        "ENGRAM_WORKSPACE": "${PWD}"
      }
    }
  }
}
```

### Cursor

`~/.cursor/mcp.json` (Cursor's documented config location):

```json
{
  "mcpServers": {
    "engram": {
      "command": "engram-mcp",
      "args": [],
      "env": {
        "ENGRAM_WORKSPACE": "/absolute/path/to/your/workspace"
      }
    }
  }
}
```

### Claude Desktop

`~/Library/Application Support/Claude/claude_desktop_config.json` (macOS) — Claude Desktop's documented location:

```json
{
  "mcpServers": {
    "engram": {
      "command": "engram-mcp",
      "args": [],
      "env": {
        "ENGRAM_WORKSPACE": "/absolute/path/to/your/workspace"
      }
    }
  }
}
```

### Important: env resolution

Engram's server reads `ENGRAM_WORKSPACE` on startup and refuses to start if the env var is unset and no `.engram/config.yaml` is found in CWD. This keeps accidental cross-workspace usage loud.

## 6. Zero-to-first-call walkthrough

Fresh machine. Target: `engram.why` returns a valid envelope within ~10 minutes of starting.

### 6.1 System prereqs (once per machine)

```bash
# Python
python --version  # expect 3.11+

# Node
node --version    # expect >=20, <24
# (Engram refuses to start on Node 24+; doc warns loudly)

# Docker (for local Milvus via compose; skip if using Zilliz Cloud)
docker --version
```

### 6.2 Install Engram

```bash
pip install engram
# pulls in serena-agent and mempalace as transitive deps
# does NOT install claude-context-mcp (Node)
```

### 6.3 Install the Node side

```bash
npm install -g @zilliz/claude-context-mcp@0.1.8
```

### 6.4 Start Milvus + Ollama for fully-local pipeline

```bash
cd engram/deploy
docker compose up -d
# Waits for milvus-standalone healthy and ollama listening
docker exec -it engram-ollama-1 ollama pull nomic-embed-text
```

`engram/deploy/compose.yaml` — bundled with Engram:

```yaml
# engram/deploy/compose.yaml
services:
  etcd:
    image: quay.io/coreos/etcd:v3.5.5
    environment:
      - ETCD_AUTO_COMPACTION_MODE=revision
      - ETCD_AUTO_COMPACTION_RETENTION=1000
      - ETCD_QUOTA_BACKEND_BYTES=4294967296
      - ETCD_SNAPSHOT_COUNT=50000
    volumes:
      - etcd_data:/etcd
    command: etcd -advertise-client-urls=http://etcd:2379 -listen-client-urls http://0.0.0.0:2379 --data-dir /etcd

  minio:
    image: minio/minio:RELEASE.2023-03-20T20-16-18Z
    environment:
      - MINIO_ACCESS_KEY=minioadmin
      - MINIO_SECRET_KEY=minioadmin
    volumes:
      - minio_data:/minio_data
    command: minio server /minio_data

  milvus-standalone:
    image: milvusdb/milvus:v2.4.9
    command: ["milvus", "run", "standalone"]
    environment:
      ETCD_ENDPOINTS: etcd:2379
      MINIO_ADDRESS: minio:9000
    depends_on: [etcd, minio]
    ports: ["19530:19530"]
    volumes:
      - milvus_data:/var/lib/milvus

  ollama:
    image: ollama/ollama:0.3.12
    volumes: [ollama_data:/root/.ollama]

  etcd_data: {}
  minio_data: {}
  milvus_data: {}
  ollama_data: {}
```

Note: the versions above are reasonable defaults; doc 10 M0 exit criterion bumps them to whichever is current at implementation time.

### 6.5 Initialize a workspace

```bash
cd ~/code/my-project
engram init --embedding-provider Ollama
# writes .engram/config.yaml
# creates .engram/anchors.sqlite with schema from src/engram/link/schema.sql
# asserts all three upstreams respond to a ping
```

### 6.6 Index the codebase

```bash
engram smoke-test
# runs the `vec.index` call end-to-end via Engram's MCP; observes it until status="completed"
```

### 6.7 Register with Claude Code and try it

```bash
claude mcp add engram -- engram-mcp
# inside a Claude Code session:
# > Use engram.why for the `process_batch` function in src/pipeline.py
```

Engram's `engram.why` runs:

1. Resolves symbol via Serena `find_symbol`.
2. Fetches anchored memories via Link Layer (empty on first run).
3. Queries MemPalace for any relevant drawers (may be empty).
4. Returns a structured envelope.

First-run memories are empty; repeated use populates them as the user's agent writes diaries / adds drawers.

## 7. Upgrade and version compatibility

- Engram locks its three upstream versions at install time (`pyproject.toml` pins + the Node global install).
- Upgrading an upstream requires bumping Engram's pins and re-testing; no silent cross-version drift.
- Engram's anchor DB schema is versioned via `PRAGMA user_version`; migrations live in `src/engram/link/migrations/`.

## 8. Uninstall

```bash
pip uninstall engram
npm uninstall -g @zilliz/claude-context-mcp
rm -rf .engram          # per-workspace anchor DB + logs
# Milvus + Ollama left running if the user wants them
```

The three upstream stores (`~/.serena`, `~/.mempalace`, `~/.context`) are untouched — Engram never writes to them outside its observed paths.

## 9. Security posture

- All three upstream WALs / DBs sit at user-only (0o600) per the inventories (01 §4, 02 §10, 03 §10).
- Engram's anchor DB inherits the same — `.engram/anchors.sqlite` created 0o600.
- No outbound network calls from Engram itself after init (beyond proxying whatever the configured embedding provider needs).
- Zilliz Cloud + OpenAI embedding providers are opt-in; the default `compose.yaml` is fully local.

## 10. Troubleshooting (first-run failures)

| Symptom | Likely cause | Fix |
|---|---|---|
| `engram init` fails: `node not found` | Node not installed or wrong version | Install Node 20.x |
| `engram init` fails: `serena start-mcp-server not found` | Python wheel not installed in active env | `pip install serena-agent==1.1.2` |
| Claude Code does not see Engram tools | `engram mcp add` registration issue | Check `~/.claude.json` for the `engram` entry |
| `engram.health` says `claude_context: last_reindex_age_seconds > 600` | Milvus or Ollama down | `docker compose ps` — restart whichever is unhealthy |
| `engram.health` says `wal_lag_seconds > 60` | WAL tailer not running or blocked on large catch-up | `engram logs` for `wal_tailer` component |
| `engram.why` returns `meta.path_degraded=no-memory` | MemPalace not reachable | check `mempalace-mcp` subprocess |

## Assumptions

- npm / npx / Node are on `PATH` when Engram starts. If not, the user gets a loud startup error, not a silent fallback.
- `pip install engram` pulls MemPalace and Serena as transitive deps; both are published to PyPI under the names used in their `pyproject.toml`.
- Docker compose versions inside `compose.yaml` are templates; M0 implementation pins to versions current at that time.
- `ENGRAM_WORKSPACE` is an acceptable convention; an alternative per-invocation `--workspace` flag is also supported for non-MCP CLI calls.
